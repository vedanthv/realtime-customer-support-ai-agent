## Classification Decision Tree (Internal Reasoning)

Ask yourself these questions in order:

**Q1: Is this asking for numbers, counts, aggregates, distribution, breakdown, or trends?**
- YES → 80% likely SQL
- Examples: "how many", "total", "top 10", "average", "demographic", "distribution"
- NO → Go to Q2

**Q2: Does it involve time-based filtering or analytics?**
- YES → 95% likely SQL
- Examples: "last month", "Q2", "this year", "weekly trend"
- NO → Go to Q3

**Q3: Is it asking for explanation, narrative, context, or "why"?**
- YES → 90% likely RAG
- Examples: "why are sales low", "explain this pattern", "what happened"
- NO → Go to Q4

**Q4: Is it a follow-up to a previous result?**
- YES → 95% likely SQL (if previous was SQL)
- Example: "what about THOSE orders" (with previous context)
- NO → Go to Q5

**Q5: Out of domain check**
- NOT in {orders, customers, payments, delivery, revenue, sales} → RAG (safe default)
- IN domain but vague → RAG (safe default, user can clarify)

## High-Confidence SQL Patterns (confidence >= 0.98)

1. **Time-series**: "trend", "over time", "last X days/weeks/months", "by month/quarter/year"
2. **Demographics**: "demographic", "breakdown", "distribution", "segment by", "cohort"
3. **Aggregations**: COUNT/SUM/AVG/MAX phrases, "total", "how many", "what's the"
4. **Ranking**: "top 10", "bottom 5", "highest", "lowest", "rank"
5. **Comparisons**: "compare", "which", "versus"
6. **Multi-dimensional**: "by region and device", "breakdown by X and Y"

## Low-Confidence Cases (should be RAG as safe default)

1. Vague: "tell me about orders", "what's going on"
2. Narrative: "why", "how", "what happened", "explain"
3. Out of scope: "best practices", "general advice"
4. Ambiguous: Could be SQL or RAG → default to RAG
